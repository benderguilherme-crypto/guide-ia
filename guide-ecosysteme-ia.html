Couche 1

IA & Agents

Les cerveaux du système. Clique sur "Exemple" pour voir un cas concret.

🧠 Modèle de langage

IA — Claude / GPT

Lit du texte, en produit, explique, analyse et génère du code. Un assistant ultra-rapide qui a toujours besoin d'une consigne.

« Rédige une page d'accueil pour un cabinet médical, ton rassurant, 500 mots, avec un appel à l'action final. »
▾ Exemple
💬 Instruction

Prompt

Le message d'instruction que tu envoies au modèle. Plus il a du contexte, un rôle, des contraintes et un format attendu, meilleure est la réponse.

❌ « Écris une page d'accueil »
✓ « Tu es copywriter. Écris une page pour un cabinet médical, ton rassurant, 500 mots, bouton Prendre rendez-vous à la fin. »
▾ Exemple
🤖 Exécuteur

Agent

Planifie, appelle des outils, garde de la mémoire et accomplit un travail en plusieurs étapes. Pas juste une réponse — une série d'actions.

« Lis ce dépôt, trouve le bug, modifie les fichiers, lance les tests, résume les changements. » — 5 étapes, 1 seule demande.
▾ Exemple
👥 Multi-agent

Squad

Une équipe d'agents spécialisés qui se passent le relais. Un agent écrit, un autre relit, un autre vérifie la qualité.

Agent « brief » → Agent « maquette » → Agent « code » → Agent « QA ». Chacun fait sa partie dans l'ordre.
▾ Exemple
🧠 La technologie derrière

LLM

LLM veut dire Large Language Model — Modèle de Langage de Grande Taille. C'est le terme technique pour désigner les IA comme Claude, GPT, Gemini. Quand tu entends LLM, pense simplement : "une IA capable de lire et produire du texte à grande échelle". Claude est un LLM. GPT est un LLM. C'est la technologie derrière tout le reste.

"J'utilise un LLM pour générer mes textes de carrousels" = "J'utilise Claude ou GPT". LLM c'est le nom de la catégorie, Claude et GPT sont des marques. Exactement comme dire "voiture" vs "Tesla".
▾ Exemple
⚙️ Prompt élaboré & sauvegardé

Skills

Un skill c'est un prompt élaboré et sauvegardé pour être réutilisé. Au lieu de réécrire tes instructions à chaque fois, tu les sauvegardes une fois comme skill — et tu l'appelles en un mot, toujours avec la même qualité. Dans les agents avancés, un skill peut aussi contenir des fichiers de référence et des règles spécifiques.

Au lieu de réécrire "Tu es expert en revue de code, vérifie la sécurité, la lisibilité et les performances, rends un rapport en 3 sections..." — tu sauvegardes ça une fois comme skill "revue de code" et tu l'appelles en un mot à chaque fois.
▾ Exemple
🔑 Coût

Tokens

Un "morceau" de texte traité par le modèle. Tout ce que tu envoies + tout ce que le modèle répond consomme des tokens. Ce n'est pas l'API qui facture — c'est le fournisseur derrière (Anthropic, OpenAI...) qui compte les tokens consommés et envoie la facture.

Tu envoies 2 000 tokens, le modèle répond 800 tokens → 2 800 tokens consommés. Surveille pour rester sous la limite et maîtriser le coût.
▾ Exemple
Couche 2

Code & Web

Les briques qui construisent ce que l'utilisateur voit et touche.

🏗 Structure

HTML

Définit la structure d'une page : titres, paragraphes, formulaires, boutons. L'ossature du bâtiment.

« ici, c'est un titre » · « ici, c'est un champ e-mail » · « ici, c'est un bouton ». Sans HTML, rien à afficher.
▾ Exemple
🎨 Apparence

CSS

Décrit la présentation visuelle : couleurs, polices, espacements, colonnes. La décoration intérieure de ta boutique.

Un bouton bleu avec des coins arrondis, un texte centré, une page en deux colonnes. Même HTML, aspect totalement différent.
▾ Exemple
⚡ Comportement

JavaScript

Ajoute le comportement dynamique : boutons qui réagissent, formulaires qui valident, données qui se chargent sans recharger la page.

La validation d'un formulaire, l'ouverture d'une fenêtre modale, le chargement d'une liste d'articles. La porte automatique.
▾ Exemple
🧩 Composants

React & Next.js

React c'est une boîte de Lego pour construire ton interface. Next.js c'est la même boîte mais avec les parties chiantes déjà pré-assemblées — pages, vitesse, référencement Google. Dans Claude Code tu ne choisis jamais : il prend Next.js automatiquement. Tu conduis, le moteur est invisible.

Tu dis à Claude Code "crée un site de réservation" — il choisit Next.js tout seul. Tu ne vois jamais le mot React. Exactement comme monter dans une voiture sans savoir si c'est essence ou diesel.
▾ Exemple
👀 Visible

Frontend

La partie visible et manipulée par l'utilisateur. La vitrine, les rayons, les boutons sur lesquels le client clique.

Page d'accueil, fiche produit, formulaire de contact, écran de réservation. Tout ce que l'œil voit.
▾ Exemple
🔒 Invisible

Backend

Ce n'est pas un outil — c'est une zone, comme la cuisine d'un restaurant. L'utilisateur ne la voit jamais mais c'est là que tout se passe : vérifier ton mot de passe, enregistrer ta commande, envoyer l'email de confirmation, activer ton abonnement. Le frontend affiche. Le backend décide, vérifie, enregistre, envoie.

Tu cliques "connexion" → le backend vérifie ton mot de passe. Tu passes une commande → le backend l'enregistre. Stripe confirme un paiement → le backend active ton abonnement. Tout ça invisible, tout ça instantané.
▾ Exemple
📦 Stockage

Base de données

Collection organisée d'informations qui survivent au redémarrage de l'application. Le grand registre de ta boutique.

Table users, table appointments, table payments. Qui sont tes clients, quels produits existent, quelles commandes ont été passées.
▾ Exemple
🗄 Clé en main

Supabase

L'endroit où vivent toutes les données de ton projet — tes utilisateurs, leurs commandes, leurs rendez-vous, leurs paiements. Quand tu fermes ton site et que tu le reouvres le lendemain, les données sont toujours là. Supabase les garde, les sécurise et les rend accessibles à ton app instantanément.

Quelqu'un s'inscrit sur ton site → son nom et email sont sauvegardés dans Supabase. Il revient 3 mois plus tard → Supabase s'en souvient. Tu fermes ton ordinateur → les données restent. C'est le registre permanent de ta boutique.
▾ Exemple
Concepts essentiels

Souvent oubliés, toujours nécessaires

Tout débutant tombe sur ces 4 concepts dans les premières heures. Sans les connaître, on bloque.

📒 La mémoire de ton projet

Git & GitHub

Git c'est la mémoire de ce que tu construis. Chaque sauvegarde s'appelle un commit. Si tu casses quelque chose, tu remontes dans la mémoire et tu repars de là où tout allait bien — comme un Ctrl+Z surpuissant qui n'oublie jamais rien, même 6 mois plus tard. GitHub c'est cette même mémoire sauvegardée en ligne — si ton ordi tombe en panne, ton projet est toujours là. Et ça existait bien avant l'IA : Git a 20 ans.

Lundi ton site marche → commit (photo). Mercredi tu casses tout → tu dis à Git "ramène-moi à lundi" → tout revient exactement comme avant. GitHub le stocke en ligne et Vercel se reconnecte automatiquement pour republier.
▾ Exemple
🔑 Sécurité

Variables d'environnement

Secrets stockés hors du code : clés API, mots de passe, clés Stripe. On ne les met JAMAIS dans le code — sinon ils se retrouvent sur GitHub, visibles de tous.

STRIPE_SECRET_KEY=sk_live_xxx · NEXT_PUBLIC_SUPABASE_URL=https://xyz.supabase.co · OPENAI_API_KEY=sk-xxx — dans un fichier .env.local que Git ignore.
▾ Exemple
🔔 Le service qui t'appelle

Webhook

Imagine que tu commandes une pizza. Sans webhook : tu appelles le restaurant toutes les 5 minutes "c'est prêt ?" "non" "c'est prêt ?" "non". Avec webhook : le restaurant t'envoie un SMS dès que la pizza sort du four. Tu ne demandes rien — on te prévient au bon moment. Stripe fait pareil : dès qu'un paiement réussit, il envoie un signal automatique à ton site qui réagit immédiatement.

Client paie → Stripe envoie un webhook → ton site reçoit le signal → active l'abonnement → envoie l'email de confirmation. Tout ça en 2 secondes, sans que tu aies rien demandé.
▾ Exemple
📦 Supermarché de code

npm

Un supermarché de briques de code disponibles sur npmjs.com — 2 millions de briques créées par des développeurs du monde entier. React, Stripe, Next.js s'installent tous via npm. Claude Code va y piocher automatiquement ce dont il a besoin, sans te demander. Tu ne touches jamais au dossier node_modules où tout est rangé. Et oui : tu peux créer ta propre brique et la publier pour que d'autres l'utilisent.

Claude Code construit ton app de réservation → il installe automatiquement Next.js, Supabase, Stripe depuis npmjs.com → tout arrive dans node_modules → tu n'ouvres jamais ce dossier. C'est le moteur sous le capot.
▾ Exemple
🔌 Connecteur IA uniquement

MCP

MCP c'est un connecteur — mais uniquement dans le monde de l'IA. Pour connecter deux logiciels entre eux on utilise une API classique — ça existe depuis 30 ans. Le MCP a été inventé avec l'ère des agents IA, pour qu'une IA puisse se brancher à un outil et comprendre ce qu'elle peut faire dedans. Les connecteurs Notion, Google Drive, Slack que tu vois dans Claude — ce sont des MCP. Tu peux créer le tien pour connecter Claude à ta propre plateforme.

Sans MCP : ton backend appelle l'API Anthropic — c'est du code qui décide. Avec MCP : tu dis à Claude "crée-moi un carrousel leadership" — Claude voit ta plateforme comme un outil disponible, la pilote tout seul, appelle les mêmes API derrière. Les API ne changent pas. Tu ajoutes juste un connecteur.
▾ Exemple
🔐 Identité

Authentification (Auth)

Mécanisme qui vérifie l'identité d'un utilisateur. Sessions, JWT, cookies — tout ça pour savoir que c'est bien toi qui te connectes.

Supabase Auth gère tout ça automatiquement. OAuth = se connecter avec Google ou GitHub plutôt qu'un mot de passe maison.
▾ Exemple
Couche finale

Publication & argent

Les couches qui rendent ton app accessible au monde et qui permettent d'en vivre.

🚀 Hébergement

Vercel

Un hébergeur automatique. Tu sauvegardes ton code sur GitHub, Vercel détecte le changement et publie ton site en quelques secondes. DNS, sécurité, serveurs — il gère tout en arrière-plan. Tu ne touches à rien.

Tu modifies une page → git push → Vercel détecte → reconstruit → publie. Ton site est à jour en moins d'une minute. Sans toucher à un seul paramètre serveur.
▾ Exemple
🌐 Adresse

Nom de domaine & DNS

L'adresse lisible par un humain : monsite.com. Le DNS traduit ce nom en adresse IP. Un domaine se loue (renouvellement annuel).

Tu achètes cabinet-dr-martin.fr chez un registrar, tu le relies à Vercel via des enregistrements DNS. 10 minutes de config, valable un an.
▾ Exemple
💳 Paiement

Stripe

Plateforme de paiement : paiements ponctuels, abonnements, parcours hébergés. Terminal + caisse + logique de facturation réunis.

Une page Checkout qui laisse un client payer ou démarrer un abonnement mensuel. Retour vers ta page de succès, mise à jour de la base de données.
▾ Exemple
👁 Tracking

Pixel Meta

Un bout de code JavaScript qu'on installe sur son site pour envoyer des données à Meta (Facebook & Instagram). Il observe ce que font les visiteurs — pages vues, clics, achats — et renvoie ces infos à Meta pour que tes publicités soient ciblées au millimètre.

Un visiteur arrive sur ton site, ajoute un produit au panier mais n'achète pas → le Pixel le signale à Meta → Meta lui reshow ta pub le lendemain sur Instagram. C'est le retargeting.
▾ Exemple
Attention

Les 5 confusions à ne plus faire

Ces erreurs de vocabulaire sont les plus fréquentes chez les débutants.

MCP = une API concurrente
→
MCP standardise la façon dont un agent découvre des outils. Ces outils peuvent eux-mêmes appeler des API.
Slack = Stack
→
Slack = espace de travail pour parler. Stack = ensemble des technologies utilisées pour coder.
Pixel = résolution d'un écran
→
Le Pixel Meta c'est un code JS installé sur ton site qui envoie des données à Facebook/Instagram pour optimiser tes pubs.
Git = GitHub
→
Git = la mémoire de ce que tu construis. GitHub = cette même mémoire sauvegardée en ligne.
API = Webhook
→
Avec une API, TU interroges. Avec un webhook, C'EST LE SERVICE qui t'appelle quand quelque chose se passe.

La checklist des 10 étapes

Clique sur chaque étape pour la cocher. C'est le chemin le plus simple pour lancer un projet web avec l'IA.

1
Rédige un prompt clair décrivant ton produit Outil : Claude / GPT
2
Laisse l'IA générer une première version Claude chat écrit le code, tu le colles dans ton éditeur — ou Claude Code génère directement les fichiers sur ton ordinateur.
3
Initialise un dépôt Git et publie-le sur GitHub Claude Code fait tout ça pour toi en une instruction — ou Claude chat te donne les commandes à taper. Toi tu as juste à créer un compte GitHub.
4
Construis le frontend et le backend Claude chat écrit le code, tu le colles — ou Claude Code le fait directement dans tes fichiers. Toi tu pilotes : tu sais où on va.
5
Stocke tes données dans une base managée Claude chat écrit le schéma et le code de connexion — ou Claude Code fait tout pour toi, y compris se connecter à Supabase directement. Toi tu as créé le projet Supabase.
6
Configure tes secrets en variables d'environnement Claude te dit exactement quoi écrire. Mais les vraies clés secrètes, c'est toi qui les colles — Claude ne doit jamais les voir.
7
Connecte GitHub à Vercel pour le déploiement automatique Tu le fais toi-même en cliquant dans Vercel — ou tu demandes à Claude Code de configurer ça pour toi. Dans les deux cas, c'est 5 minutes.
8
Relie un nom de domaine lisible Tu enregistres le domaine toi-même (un humain doit payer 😄) — Claude Code configure les DNS et relie tout à Vercel.
9
Intègre Stripe pour encaisser Claude chat écrit le code d'intégration — ou Claude Code le fait directement dans ton projet. Toi tu as créé le compte Stripe et tu as les clés.
10
Configure les webhooks Stripe pour réagir aux événements Claude chat ou Claude Code écrit le code qui reçoit les événements. La configuration finale dans le dashboard Stripe : tu la fais manuellement, ou tu demandes à Claude Code.
